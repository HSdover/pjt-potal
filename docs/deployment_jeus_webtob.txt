Governance Portal WebtoB + JEUS 배포 구성
작성일: 2026-04-28
수정일: 2026-05-06

1. 전제

- 운영 WAS는 JEUS 9 기준으로 구성합니다.
- 백엔드는 Spring Boot 3.3.7 기반이며 `jakarta.*` 패키지 체계를 사용합니다.
- 프론트엔드는 Vue 3/Vite 빌드 산출물을 Spring Boot WAR 안의 정적 리소스로 포함합니다.
- 운영 서버에서 Node.js 서버를 띄우지 않습니다. Node.js/npm은 WAR 빌드 중 프론트엔드 빌드 단계에서만 사용합니다.
- `.\gradlew.bat bootWar` 실행 시 Gradle Node Plugin이 지정된 Node/npm을 준비하고 `npm install`, `npm run build`를 실행합니다.

2. 운영 서버 구성

```text
사용자 브라우저
  -> WebtoB
     - HTTPS/HTTP 진입점
     - 서비스 요청을 JEUS로 전달
  -> JEUS 9
     - governance-portal-backend.war 배포
     - Vue 정적 파일 제공
     - Spring Boot REST API 처리
     - Spring Security, JPA/MyBatis, Redis, 외부 API 연동 기반 제공
  -> DB / Redis / IAM / Dataiku / OCI 등 외부 시스템
```

3. URL 라우팅 예시

- `https://governance.example.com/`
  - WebtoB가 JEUS로 전달하고 Spring Boot가 Vue `index.html`을 반환합니다.
- `https://governance.example.com/assets/*`
  - WebtoB가 JEUS로 전달하고 Spring Boot가 Vue 빌드 정적 파일을 반환합니다.
- `https://governance.example.com/api/catalog/metadata`
  - Spring Boot REST API가 메타데이터 카탈로그를 반환합니다.
- `https://governance.example.com/api/catalog/lineage`
  - Spring Boot REST API가 리니지 흐름을 반환합니다.
- `https://governance.example.com/api/catalog/source-sample`
  - Spring Boot REST API가 HIRA 원천 샘플을 반환합니다.
- `https://governance.example.com/swagger-ui.html`
  - 필요 시 WebtoB가 JEUS의 Swagger UI 경로로 전달합니다.

4. Frontend 빌드 산출물

- 작업 경로: `governance-portal\frontend`
- 수동 빌드 명령:

```powershell
cd frontend
npm install
npm run build
```

- 운영 WAR 빌드에서는 별도 수동 프론트 빌드가 필수는 아닙니다.
- `backend`에서 `.\gradlew.bat bootWar`를 실행하면 Gradle이 프론트 빌드를 자동 수행합니다.
- 산출물 위치: `governance-portal\frontend\dist`
- WAR 포함 위치: `backend\build\generated\frontend\static`

5. Backend 배포 산출물

- 작업 경로: `governance-portal\backend`
- 빌드 명령:

```powershell
.\gradlew.bat bootWar
```

- 최종 산출물: `governance-portal\backend\build\libs\governance-portal-backend.war`
- JEUS에는 WAR 파일 하나를 애플리케이션으로 배포합니다.
- Context Path는 운영 정책에 따라 `/` 또는 `/governance` 등으로 지정할 수 있습니다.
- WebtoB 전달 경로와 JEUS Context Path는 반드시 일치해야 합니다.

6. 현재 프로젝트의 배포 관련 구현

- `backend\build.gradle`
  - `war` 플러그인을 사용합니다.
  - `bootWar.archiveFileName`은 `governance-portal-backend.war`로 고정되어 있습니다.
  - `bootJar`는 비활성화되어 있습니다.
  - 내장 Tomcat은 `providedRuntime`으로 처리해 외부 Servlet 컨테이너와 중복되지 않도록 합니다.
  - Gradle Node Plugin으로 프론트엔드 `npm install`, `npm run build`를 자동 실행합니다.
  - `frontend\dist`를 `backend\build\generated\frontend\static`으로 복사하고 WAR 정적 리소스에 포함합니다.
- `backend\src\main\java\com\example\governanceportal\GovernancePortalApplication.java`
  - `SpringBootServletInitializer`를 상속해 외부 Servlet 컨테이너에서 WAR가 초기화되도록 합니다.
- `backend\src\main\java\com\example\governanceportal\config\SpaForwardConfig.java`
  - API가 아닌 프론트엔드 라우트에서 새로고침했을 때 `index.html`로 forward합니다.
- `backend\src\main\java\com\example\governanceportal\config\SecurityConfig.java`
  - 현재 개발 단계에서는 정적 리소스, Swagger, H2 console, `/api/**` 접근을 허용합니다.

7. 개발 환경과 운영 환경 차이

- 개발 환경
  - Frontend: Vite dev server, `http://localhost:5173`
  - Backend: IntelliJ 또는 Gradle 실행, `http://localhost:8080`
  - Vite가 `/api` 요청을 백엔드로 프록시합니다.
- 운영 환경
  - Frontend: Spring Boot WAR 내부 정적 리소스
  - Backend: 같은 Spring Boot WAR 내부 REST API
  - Web/WAS: WebtoB -> JEUS 9 -> `governance-portal-backend.war`

8. 주의 사항

- JEUS 9와 Spring Boot 3.x의 Jakarta EE 호환성을 실제 운영 표준에서 확인해야 합니다.
- 운영에서는 JDBC DataSource, Redis 접속 정보, IAM/OIDC 설정을 `application.yml` 또는 외부 설정으로 분리해야 합니다.
- 운영 DB를 붙일 때는 `GOVERNANCE_DATASOURCE_URL`, `GOVERNANCE_DATASOURCE_USERNAME`, `GOVERNANCE_DATASOURCE_PASSWORD`를 외부 환경 변수로 지정합니다.
- 운영에서는 샘플 스키마 초기화를 피하기 위해 `GOVERNANCE_SQL_INIT_MODE=never`를 명시하는 것이 안전합니다.
- 운영에서는 샘플 CSV 자동 적재를 피하기 위해 `GOVERNANCE_SAMPLE_DATA_ENABLED=false`를 명시합니다.
- 운영에서는 `GOVERNANCE_H2_CONSOLE_ENABLED=false`를 명시합니다.
- 현재 `SecurityConfig`는 개발 편의를 위해 `/api/**`와 `anyRequest()`를 허용합니다. 운영 전 인증/인가 정책을 반영해야 합니다.
- Vue SPA 새로고침 지원은 `SpaForwardConfig`가 담당합니다.
- `/api/*`, `/api-docs/*`, `/swagger-ui/*`, `/actuator/*`, `/h2-console/*` 경로는 SPA forward 대상에서 제외됩니다.
- Context Path를 `/governance` 등으로 운영할 경우 Vite base path, WebtoB 전달 경로, Spring Boot 정적 리소스 경로를 함께 점검해야 합니다.
