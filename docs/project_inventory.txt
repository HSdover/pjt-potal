Governance Portal 프로젝트 구성 문서

작성일: 2026-05-11

1. 프로젝트 개요

Governance Portal은 Vue 3 프론트엔드와 Spring Boot 3.3 백엔드로 구성된 데이터 거버넌스 포털 샘플입니다.

운영 배포는 Nginx + Spring Boot executable JAR 구조를 기준으로 합니다.

- Nginx: Vue 정적 파일 서빙, API reverse proxy
- Spring Boot: embedded Tomcat 기반 API 서버
- Frontend 산출물: `frontend/dist`
- Backend 산출물: `backend/build/libs/governance-portal-backend.jar`

2. 주요 디렉터리

- `backend`
  - Java 21, Spring Boot 3.3 기반 백엔드 프로젝트입니다.
  - 실행 JAR를 생성합니다.
  - REST API, H2 개발 DB, Spring Security, OpenAPI, Actuator 설정을 포함합니다.
- `frontend`
  - Vue 3, TypeScript, Vite 기반 프론트엔드 프로젝트입니다.
  - 개발 중에는 Vite dev server를 사용하고, 운영에서는 `frontend/dist` 정적 파일을 Nginx가 제공합니다.
- `infra`
  - Nginx와 systemd 예시 설정을 둡니다.
- `docs`
  - 프로젝트 구조, 배포, 오프라인 빌드 문서를 둡니다.
- `offline`
  - 내부망 오프라인 빌드를 위한 npm/Gradle cache를 둡니다.
- `scripts`
  - 운영 빌드와 오프라인 빌드 스크립트를 둡니다.
- `data`
  - HIRA 원천/가공 샘플 데이터를 둡니다.
- `개선사항`
  - 프론트엔드 템플릿화와 개선 후보를 기록합니다.

3. Backend 주요 파일

- `backend/build.gradle`
  - Java 21 toolchain, Spring Boot 3.3.7, 실행 JAR 빌드를 정의합니다.
  - `bootJar` 산출물 이름은 `governance-portal-backend.jar`입니다.
  - Vue 빌드 결과를 JAR에 포함하지 않습니다.
- `backend/src/main/resources/application.yml`
  - 서버 포트 기본값은 `18080`입니다.
  - reverse proxy 환경을 위해 `server.forward-headers-strategy=framework`를 사용합니다.
- `backend/src/main/resources/schema.sql`
  - `hira_institution_sample`, `metadata_catalog`, `lineage_flow` 테이블을 생성합니다.
- `backend/src/main/resources/data.sql`
  - 샘플 메타데이터와 리니지 데이터를 적재합니다.
- `backend/src/main/java/com/example/governanceportal/GovernancePortalApplication.java`
  - 실행 JAR 기준 Spring Boot 진입점입니다.
- `backend/src/main/java/com/example/governanceportal/catalog/CatalogController.java`
  - 현재 프론트엔드가 사용하는 샘플 REST API를 제공합니다.
  - `GET /api/catalog/metadata`
  - `GET /api/catalog/lineage`
  - `GET /api/catalog/source-sample`

4. Frontend 주요 파일

- `frontend/package.json`
  - Vite/Vue 개발, 빌드, 테스트 스크립트를 정의합니다.
- `frontend/vite.config.ts`
  - 개발 중 `/api` 요청을 기본적으로 `http://localhost:18080`으로 프록시합니다.
- `frontend/src/router/index.ts`
  - 주요 화면 라우트를 정의합니다.
- `frontend/src/features`
  - SI 템플릿화 기준에 맞춰 업무 기능별 API, 타입, 컬럼, 화면을 둡니다.
- `frontend/src/shared`
  - 공통 API 클라이언트, 목록 타입, 그리드, 검색 패널, 권한 버튼을 둡니다.
- `frontend/src/components/LineageFlow.vue`
  - Vue Flow 기반 리니지 그래프 컴포넌트입니다.
- `frontend/src/components/GridPageLayout.vue`
  - 목록형 화면 공통 레이아웃입니다.

5. Infra 주요 파일

- `infra/nginx/governance-portal.conf`
  - Vue 정적 파일 서빙과 Spring Boot API reverse proxy 예시입니다.
- `infra/systemd/governance-portal.service`
  - Spring Boot 실행 JAR systemd 서비스 예시입니다.

6. Scripts

- `scripts/build-production.ps1`
  - 프론트엔드 정적 파일과 백엔드 실행 JAR를 함께 빌드합니다.
- `scripts/prepare-offline-bundle.ps1`
  - 외부망 PC에서 오프라인 빌드 캐시를 준비합니다.
- `scripts/build-offline-jar.ps1`
  - 내부망 PC에서 오프라인 모드로 정적 파일과 JAR를 빌드합니다.

7. 개발 실행

Backend:

```powershell
cd backend
.\gradlew.bat bootRun
```

Frontend:

```powershell
cd frontend
npm install
npm run dev
```

8. 운영 빌드

```powershell
.\scripts\build-production.ps1
```

9. 배포 기준

- `frontend/dist`는 Nginx 정적 파일 루트에 배치합니다.
- `governance-portal-backend.jar`는 Spring Boot API 서버로 systemd에서 실행합니다.
- Nginx가 `/api/*`, `/api-docs/*`, `/swagger-ui*`, `/actuator/*`를 Spring Boot로 전달합니다.
