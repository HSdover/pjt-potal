Governance Portal 프로젝트 구성 문서

작성일: 2026-05-11
갱신일: 2026-05-12

1. 프로젝트 개요

Governance Portal은 Vue 3 프론트엔드와 Spring Boot 3.3 백엔드로 구성된 데이터 거버넌스 포털 샘플입니다.

운영 배포는 Nginx + Spring Boot executable JAR 구조를 기준으로 합니다.

- Nginx: Vue 정적 파일 서빙, API reverse proxy
- Spring Boot: embedded Tomcat 기반 API 서버
- Frontend 산출물: `frontend/dist`
- Backend 산출물: `backend/build/libs/governance-portal-backend.jar`

2. 주요 디렉터리

- `backend`
  - Java 21, Spring Boot 3.3 기반 백엔드 프로젝트입니다.
  - REST API, H2 개발 DB, Spring Security, OpenAPI, Actuator 설정을 포함합니다.
- `frontend`
  - Vue 3, TypeScript, Vite 기반 프론트엔드 프로젝트입니다.
  - 개발 중에는 Vite dev server를 사용하고, 운영에서는 `frontend/dist` 정적 파일을 Nginx가 제공합니다.
- `tools/generator`
  - 표준 목록 화면 feature 파일을 생성하는 로컬 개발 도구와 페이지 spec을 둡니다.
- `infra`
  - Nginx와 systemd 예시 설정을 둡니다.
- `docs`
  - 프로젝트 구조, 배포, 오프라인 빌드, 프론트엔드 온보딩 문서를 둡니다.
- `offline`
  - 내부망 오프라인 빌드를 위한 npm/Gradle cache를 둡니다.
- `scripts`
  - 운영 빌드와 오프라인 빌드 스크립트를 둡니다.
- `data`
  - HIRA 원천/가공 샘플 데이터를 둡니다.
- `개선사항`
  - 프론트엔드 템플릿화 기준, 완료 내역, 후속 개선 후보를 기록합니다.

3. Backend 주요 파일

- `backend/build.gradle`
  - Java 21 toolchain, Spring Boot 3.3.7, 실행 JAR 빌드를 정의합니다.
  - `bootJar` 산출물 이름은 `governance-portal-backend.jar`입니다.
  - Vue 빌드 결과를 JAR에 포함하지 않습니다.
- `backend/src/main/resources/application.yml`
  - 서버 포트 기본값은 `18080`입니다.
  - reverse proxy 환경을 위해 `server.forward-headers-strategy=framework`를 사용합니다.
- `backend/src/main/resources/application-local.yml`
  - 로컬 백엔드 시작 시 프론트 dev server를 빌드 후 재시작하도록 설정합니다.
- `backend/src/main/resources/schema.sql`
  - `hira_institution_sample`, `metadata_dataset`, `lineage_flow` 테이블을 생성합니다.
- `backend/src/main/resources/data.sql`
  - 샘플 메타데이터와 리니지 데이터를 적재합니다.
- `backend/src/main/java/com/example/governanceportal/GovernancePortalApplication.java`
  - 실행 JAR 기준 Spring Boot 진입점입니다.
- `backend/src/main/java/com/example/governanceportal/metadata`
  - 메타데이터 API, 서비스, repository, DTO를 둡니다.
  - `GET /api/metadata`
  - `POST /api/metadata/search`
- `backend/src/main/java/com/example/governanceportal/sourcesample`
  - 원천 샘플 API, 서비스, repository, DTO를 둡니다.
  - `GET /api/source-sample`
  - `POST /api/source-sample/search`
- `backend/src/main/java/com/example/governanceportal/lineage`
  - 리니지 API, 서비스, repository, DTO를 둡니다.
  - `GET /api/lineage`
- `backend/src/main/java/com/example/governanceportal/common/list`
  - `ListResponse`, `ListSortRequest`, `PageSupport` 같은 목록 공통 타입과 유틸을 둡니다.
- `backend/src/main/java/com/example/governanceportal/sampledata`
  - 로컬 샘플 데이터 초기화 코드를 둡니다.
- `backend/src/main/java/com/example/governanceportal/localdev`
  - 로컬 프론트 dev server 빌드/재시작 연동 코드를 둡니다.
- `backend/src/test/java/com/example/governanceportal/GovernancePortalApplicationTests.java`
  - 카탈로그 API와 목록 조회 계약을 검증합니다.

4. Frontend 주요 파일

- `frontend/package.json`
  - Vite/Vue 개발, 빌드, 테스트 스크립트를 정의합니다.
- `tools/generator/generate-feature.cmd`
  - `generator.config.json`을 읽어 터미널 인자 없이 표준 목록 화면과 기본 백엔드 패키지를 생성합니다.
- `frontend/vite.config.ts`
  - 개발 중 `/api` 요청을 기본적으로 `http://localhost:18080`으로 프록시합니다.
- `frontend/src/router/index.ts`
  - 라우트, 메뉴 노출 여부, 메뉴 순서, 화면 권한 코드를 `meta` 기준으로 관리합니다.
- `frontend/src/App.vue`
  - 라우터 `meta.menu` 기준으로 메뉴를 렌더링합니다.
- `frontend/src/features/metadata`
  - 메타데이터 목록+상세 기준 화면입니다.
- `frontend/src/features/source-sample`
  - 원천 샘플 목록 화면입니다.
- `frontend/src/features/lineage`
  - 데이터 리니지 그래프와 처리 단계 화면입니다.
- `frontend/src/shared/api`
  - 공통 HTTP 클라이언트와 API 오류 타입을 둡니다.
- `frontend/src/shared/types/list.ts`
  - `ListRequest`, `ListResponse`, `ListSort` 표준 타입을 둡니다.
- `frontend/src/shared/components/grid/BaseGrid.vue`
  - AG Grid 공통 래퍼입니다.
- `frontend/src/shared/components/search/SearchPanel.vue`
  - 검색 조건과 액션 버튼 공통 배치 컴포넌트입니다.
- `frontend/src/shared/components/auth/AuthButton.vue`
  - 샘플 권한 기준 버튼 노출 제어 컴포넌트입니다.
- `frontend/src/views/DashboardView.vue`
  - 대시보드 화면입니다.
- `frontend/src/views/ForbiddenView.vue`
  - 라우트 권한이 없을 때 표시하는 fallback 화면입니다.

6. Infra 주요 파일

- `infra/nginx/governance-portal.conf`
  - Vue 정적 파일 서빙과 Spring Boot API reverse proxy 예시입니다.
- `infra/systemd/governance-portal.service`
  - Spring Boot 실행 JAR systemd 서비스 예시입니다.

7. Scripts

- `scripts/build-production.ps1`
  - 프론트엔드 정적 파일과 백엔드 실행 JAR를 함께 빌드합니다.
- `scripts/prepare-offline-bundle.ps1`
  - 외부망 PC에서 오프라인 빌드 캐시를 준비합니다.
- `scripts/build-offline-jar.ps1`
  - 내부망 PC에서 오프라인 모드로 정적 파일과 JAR를 빌드합니다.

8. 개발 실행

Backend:

```powershell
cd backend
.\gradlew.bat bootRun
```

Frontend:

```powershell
cd frontend
npm install
npm run dev
```

9. 운영 빌드

```powershell
.\scripts\build-production.ps1
```

10. 배포 기준

- `frontend/dist`는 Nginx 정적 파일 루트에 배치합니다.
- `governance-portal-backend.jar`는 Spring Boot API 서버로 systemd에서 실행합니다.
- Nginx가 `/api/*`, `/api-docs/*`, `/swagger-ui*`, `/actuator/*`를 Spring Boot로 전달합니다.
