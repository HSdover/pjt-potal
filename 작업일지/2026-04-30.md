# 2026-04-30 작업일지

오늘은 프론트 화면 구조를 `View.vue + View.ts` 1:1 방식으로 정리했다. 각 `.vue`는 화면 상태 연결, 이벤트 바인딩, 템플릿을 담당하고, 대응 `.ts`는 타입, AG Grid 컬럼 정의, 화면 전용 API 호출, helper를 담당한다.

`catalog store`는 현재 화면 간 데이터 공유가 없어 제거했고, `MetadataView`, `SourceSampleView`, `LineageView`가 각자 필요한 API를 직접 호출하도록 변경했다. 단, Pinia 자체는 향후 로그인/권한/공통 알림 같은 전역 상태를 위해 유지했다.

JSP 샘플은 `TableDetailSampleView.vue/.ts`로 변환했고, `common-style.css`는 `frontend/public/styles` 정적 리소스로 배치했다.

개선사항 문서에서는 Pinia 제거 항목을 현재 방향에 맞게 수정했고, fetch 실패 시 사용자 피드백이 없는 항목은 남은 개선사항으로 유지했다.

검증: `npm.cmd run build`, `.\gradlew.bat test` 통과.
