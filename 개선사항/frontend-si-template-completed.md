# SI 프로젝트 프론트엔드 템플릿 기준 요약

작성일: 2026-05-12
갱신일: 2026-05-19

이 문서는 프론트엔드 템플릿화 기준 중 현재 참고할 내용만 남긴 요약 문서다.
현재 기능목록 엑셀 기준 화면 도메인/템플릿 분류의 실행 기준은 `docs/governance/screen-domain-template.md`다.

## 현재 기준

- 현재 단계에서는 프론트엔드 구현 기준을 추가로 확정하지 않는다.
- 기능목록 기준 화면 도메인 분류는 `docs/governance/screen-domain-template.md`에서 관리한다.
- 프로젝트가 실제로 시작되면 이 문서의 기준을 참고해 구현 세부 기준을 다시 정한다.

## 참고 가능한 기준

- sample-list, sample-list-jpa는 개발 참고 화면으로 둔다.
- 목록/검색/등록/수정 흐름은 sample-list 화면을 참고할 수 있다.
- 화면 제목과 메뉴 표시는 라우터 meta 기준을 참고할 수 있다.
- 반복 화면 생성이 필요해지면 통합 생성기 사용 여부를 다시 검토한다.
- 프론트엔드 온보딩 문서는 개발 착수 후 참고한다.

## 현재 보류되는 판단

아래 판단은 이번 화면 도메인 분류 단계에서 확정하지 않는다.

- 신규 업무 화면의 실제 파일 구조
- 목록 API 계약
- 화면별 공통 컴포넌트 적용 기준
- 라우트 권한 기준
- 로그인/사용자별 권한 API 기준
- 빌드/테스트 품질 게이트 기준

## 다시 볼 시점

이 문서는 다음 단계에서 다시 참고한다.

1. 화면 후보가 확정됨
2. 상세 화면 스펙을 작성하기로 결정됨
3. 프론트엔드 구현 기준을 정해야 함
4. 프로젝트 개발 착수 일정이 확정됨
