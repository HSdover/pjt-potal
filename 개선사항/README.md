# 개선사항 정리

프로젝트 진행 중 발견되는 개선사항과 작업 방향을 기록한다.

현재 기준의 핵심은 **기능목록 엑셀 기준 화면 도메인/템플릿 분류**다.
프론트엔드, 백엔드, DB, 외부 연계 로직은 프로젝트 시작 이후 개발 단계에서 다룬다.

## 문서 목록

- `작업방향.md`: 현재 작업 원칙, 하지 않을 것, 사용자 검토 순서
- `frontend-si-template-completed.md`: 프론트엔드 템플릿 기준 요약
- `../docs/governance/screen-domain-template.md`: 기능목록 엑셀 기준 화면 도메인/템플릿 분류
- `../docs/governance/project-start-strategy.md`: 프로젝트 투입 후 내부망 형상관리/Oracle Linux 운영 배포 전략

## 현재 우선순위

1. 기능목록 엑셀의 A01~G02 항목을 화면 후보로 정리
2. 화면 후보를 상위 도메인으로 분류
3. 화면 후보를 템플릿 유형으로 분류
4. 독립 화면/탭/섹션 후보 표시
5. 프로젝트 시작 전까지 구현 설계 문서는 보류

## 프로젝트 투입 후 후속 우선순위

1. 내부망 형상관리 정책 확인
2. 내부 Git 저장소/브랜치/릴리즈 전략 수립
3. OCI Oracle Linux 운영 서버 배포 구조 확정
4. 배포, 백업, 롤백, 로그 관리 절차 수립
